# 配置管理系统测试说明

## 测试内容

本测试套件包含了以下配置对象的单元测试：

1. **系统配置对象测试**（SystemConfigTests）
   - 数据库连接配置测试
   - 系统路径配置测试
   - 启动配置测试

2. **界面配置对象测试**（UIConfigTests）
   - 布局配置测试
   - 主题配置测试
   - 字体配置测试

3. **多媒体配置对象测试**（MediaConfigTests）
   - 图片背景配置测试
   - 抽屉面板配置测试
   - 视频片段配置测试

## 运行测试的方法

### 方法一：使用批处理脚本（推荐）

1. 双击运行 `RunTests.bat` 批处理文件
2. 批处理文件会自动编译测试运行器并在控制台模式下运行所有测试
3. 测试结果将在控制台窗口中显示
4. 测试完成后，按任意键关闭窗口

### 方法二：手动编译和运行

1. 打开Delphi IDE
2. 打开项目文件 `TestRunner.dproj`
3. 编译项目（Shift+F9）
4. 运行测试：
   - 不带参数运行：将打开图形界面运行器
   - 带参数 `console` 运行：在控制台模式下运行

## 测试输出解释

测试输出将包含以下信息：

- 测试套件名称和测试数量
- 每个测试的执行结果（成功/失败）
- 失败测试的详细错误信息
- 测试执行时间统计
- 总体测试结果摘要

## 添加新测试

如需添加新的测试，请遵循以下步骤：

1. 创建新的测试单元（XXXTests.pas）
2. 在该单元中定义测试类（继承自TTestCase）
3. 实现测试方法（使用Check...断言方法）
4. 在initialization部分注册测试套件
5. 在TestRunner.dpr中引用新的测试单元

## 故障排除

如果遇到测试运行问题，请检查：

1. Delphi编译器路径是否正确设置（在RunTests.bat中）
2. DUnit框架是否可用
3. 所有依赖单元是否存在并且能够被编译器找到
4. 测试过程中使用的资源文件是否存在

## 注意事项

- 测试会创建临时文件和目录，测试结束后会自动清理
- 某些测试可能需要特定的系统权限才能运行（如文件读写权限）
- 测试使用独立的测试目录，不会影响实际的配置文件
