# ConfigBuild 属性类型参考表

本文档列出了ConfigBuild支持的所有属性类型，按照使用频率从高到低排序。包括已实现的类型和计划实现的类型。

## 属性类型选择指南

在选择属性类型时，请考虑以下因素：

1. **数据复杂度**：简单的单一值使用简单属性类型，多个相关值组合使用复杂属性类型
2. **查询频率**：经常查询的配置使用简单属性类型，提高访问速度
3. **修改频率**：频繁修改的配置考虑使用简单属性类型，降低复杂度
4. **结构化需求**：需要结构化数据的情况使用复杂属性类型
5. **特殊格式**：需要特定验证或格式化的数据使用专用属性类型

## 扩展自定义属性类型

ConfigBuild支持扩展自定义属性类型。您可以通过以下步骤添加新的属性类型：

1. 在ConfigTypes.pas中扩展TConfigType枚举
2. 创建对应的编辑器类
3. 实现Load和Save方法处理数据转换
4. 在配置类型注册表中注册新类型

请参考开发文档了解更多关于自定义属性类型的信息。
