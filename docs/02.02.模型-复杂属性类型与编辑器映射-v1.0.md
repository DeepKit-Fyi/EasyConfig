# 复杂属性类型与编辑器映射

> 本文描述 ConfigBuild 中“复杂属性”的类型体系，以及它们与 FMX Frame 编辑器之间的映射关系和扩展方式。

## 1. 概述

- 简单属性（文本、整数、浮点、布尔、颜色、路径等）通过网格或原生控件直接编辑；
- 复杂属性（字体、数据库连接、背景布局、视频片段等）需要专用编辑器；
- 每个复杂属性在 JSON 中有清晰的结构，并由一个 `TBaseConfigFrame` 子类负责可视化编辑。

## 2. 复杂属性类型一览（优先支持）

以下类型在 MVP/近期版本中被优先支持（名称示意，具体以 `UtilsTypes.pas` 为准）：

- 字体相关：
  - `Font`：字体名称、字号、样式、颜色等。

- 数据库与 API：
  - `Database`：数据库连接字符串、主机、端口、用户名等；
  - `AIAPI`：大模型 API 的地址、Key、Model 名称等（敏感字段须结合 UniBase.Security）。

- 背景与图层：
  - `BgDraw`：背景底图、颜色、尺寸；
  - `TextOnBg`：叠加在背景上的文本内容、位置、字体；
  - `ImageOnBg`：叠加在背景上的图片、透明度、缩放模式等。

- 视频相关：
  - `VideoClip`：单个视频片段的路径、起止时间、音量；
  - `Video`：由多个 `VideoClip` 组合而成的整体视频配置。

## 3. 类型枚举与 Frame 映射

### 3.1 类型系统

- 在 `UtilsTypes.pas` 中定义：
  - `TConfigType`：配置项的逻辑类型（简单/复杂）；
  - `TEditorType`：对应的编辑器类型（简单编辑控件 / 复杂 Frame）；
  - 复杂属性类型枚举，例如：`ctFont`, `ctDatabase`, `ctAIAPI`, `ctBgDraw`, `ctVideoClip` 等。

### 3.2 Frame 映射规则

- 所有复杂属性编辑器继承自同一基类，例如：`TBaseConfigFrame`：
  - 公开方法：`LoadFromJSON(AObject)` / `SaveToJSON(AObject)`；
  - 通过属性/方法与主界面或 Controller 通信。

- `ControllerConfigs.pas` 中维护一个“类型 → Frame 类”的映射表：

```pascal
case AConfigType of
  ctFont:
    Result := TFrameFontEditor;
  ctDatabase:
    Result := TFrameDBEditor;
  ctAIAPI:
    Result := TFrameAIAPIEditor;
  ctBgDraw:
    Result := TFrameBgDrawEditor;
  ctVideoClip:
    Result := TFrameVideoClipEditor;
end;
```

- Controller 负责：
  - 根据当前选中属性的类型创建对应 Frame；
  - 把 JSON 子对象传入 `LoadFromJSON`；
  - 在保存时调用 `SaveToJSON` 并回写到整体 JSON。

## 4. JSON 结构示例（概念级）

> 以下示例仅说明结构，不代表最终字段名和完整属性。

### 4.1 字体配置

```json
{
  "_id": "font_main",
  "_type": "Font",
  "family": "Segoe UI",
  "size": 14,
  "style": ["bold"],
  "color": "#333333"
}
```

### 4.2 数据库连接

```json
{
  "_id": "db_default",
  "_type": "Database",
  "host": "127.0.0.1",
  "port": 5432,
  "database": "appdb",
  "user": "appuser",
  "password_ref": "secret:db_default_password"
}
```

- 密码字段建议仅保存引用（如 `password_ref`），真实值通过 UniBase.Security 获取。

### 4.3 视频片段

```json
{
  "_id": "clip_intro",
  "_type": "VideoClip",
  "file": "videos/intro.mp4",
  "start_sec": 0,
  "end_sec": 10,
  "volume": 0.8
}
```

## 5. 新增复杂属性类型的步骤

1. **类型定义**：
   - 在 `UtilsTypes.pas` 中新增对应的复杂属性类型枚举值；
   - 更新任何需要的帮助函数（如类型到名称的映射）。

2. **JSON 设计**：
   - 在本文件或相关模型文档中，设计好 JSON 字段结构；
   - 确保字段名清晰、易于验证与扩展。

3. **编辑器 Frame 实现**：
   - 新建一个 FMX Frame，继承自 `TBaseConfigFrame`；
   - 实现 `LoadFromJSON` / `SaveToJSON`；
   - 添加必要的校验逻辑和人机工程（提示、默认值等）。

4. **在 ControllerConfigs 中注册映射**：
   - 将新类型与新 Frame 类关联起来；
   - 确保在 UI 触发时能够正确创建该 Frame。

5. **测试与文档**：
   - 添加至少一个包含新类型的示例配置；
   - 增补测试用例（或手工测试步骤）；
   - 在产品/使用文档中提及新类型的用途和限制。

## 6. 未来扩展：自定义类型

- `UtilsTypes.pas` 中可以预留 `ctCustom` / `etCustom` 等枚举值；
- 通过注册机制允许外部模块在不修改核心代码的情况下：
  - 注册新的复杂属性类型；
  - 提供对应的 FMX Frame 实现；
  - （可选）登记验证规则和帮助文档链接；
- 具体插件 / 扩展点设计可在未来版本单独文档中描述。
