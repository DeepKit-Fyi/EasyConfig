# 通用配置编辑器操作手册

> 本文面向使用 ConfigBuild（通用配置编辑器）的开发者和运维人员，
> 介绍如何打开、浏览、编辑、验证和保存 INI+JSON 配置。

## 1. 概述

- ConfigBuild 用于编辑某个工程的 INI + JSON 配置对（例如：`app_config.ini` + `app_config.json`）。
- 本工具不会执行业务逻辑，只负责生成和修改配置文件。

## 2. 启动应用

1. 双击应用程序可执行文件启动 ConfigBuild；
2. 初次启动时，工具会使用 UniBase 初始化自身配置和日志环境；
3. 主界面大致分为两大 Tab：
   - **配置编辑**：编辑当前打开的 INI+JSON 工程（默认 Tab）；
   - **文件浏览**：浏览磁盘上的 INI/JSON 文件、批量增删和预览；
4. 在"配置编辑"Tab 下，界面进一步划分为：
   - 左侧：配置文件 / 节点树；
   - 中间上：结构视图（属性网格）与"原始文档"子视图；
   - 中间下：复杂属性编辑器区域；
   - 右侧：属性编辑器（类似 Delphi 对象检查器）+ 类型工具箱；
   - 底部：状态栏 + 日志 / 验证结果输出。
5. 状态栏位于主界面最底部，显示：
   - 当前文件路径和修改状态（已修改 / 已保存）；
   - 验证结果摘要（如"3 个错误 / 2 个警告"）；
   - 最近操作提示。

## 3. 打开配置工程

1. 在菜单或工具栏中选择“打开配置”按钮；
2. 在文件对话框中选择一个 INI 文件（例如 `app_config.ini`）；
3. 工具会：
   - 解析 INI 的 `[json_file]` 节，自动寻找 JSON 文件路径；
   - 加载 JSON 文件并构建内部模型；
   - 在左侧树和中间视图中展示结构和内容。

> 若 JSON 文件不存在，工具可根据约定创建一个空的 JSON 文件（具体行为以实现为准）。

## 3.1 配置编辑 Tab 与原始文档视图

在“配置编辑”Tab 中，主要有两个视角：

- **结构视图**：
  - 默认视图，左侧是树，右侧是属性网格和复杂属性编辑器；
  - 适合做结构化增删改查和规则验证。
- **原始文档视图**：
  - 通过中间的子 Tab 切换到“原始文档”；
  - 左侧仍然是当前工程树，右侧分为“INI 原文 / JSON 原文”两个子 Tab，使用文本组件显示完整内容；
  - 用于查看或（在允许时）直接编辑底层 INI / JSON 文本。

## 4. 浏览配置

### 4.1 左侧树

- 左侧树展示当前工程中 INI 和 JSON 的逻辑结构，例如：
  - INI 的节和键；
  - JSON 的对象树和数组项；
- 单击树节点，可在中间区域和右侧显示对应内容与属性。

### 4.2 INI / JSON 内容视图

- 中间上方区域通常包含两个 Tab：
  - “INI 文件”Tab：以网格形式展示“属性名 / 属性类型 / 属性值”；
  - “JSON 文件”Tab：以树 + 文本预览（或纯文本）展示 JSON 结构。
- 底层 `TMemo` 可用于查看或编辑原始 INI / JSON 文本（根据权限控制）。

## 5. 编辑简单属性

1. 在左侧树或网格中选中某个简单属性（例如：字符串、整数、布尔值等）；
2. 在网格中直接修改其值，或使用右侧属性面板中的控件编辑：
   - 文本：`TEdit`；
   - 整数 / 浮点：带范围校验的输入框；
   - 布尔：`TCheckBox`；
   - 颜色：颜色选择器控件；
   - 路径：带“浏览…”按钮的路径编辑控件。
3. 修改后，视图会同步更新对应的 INI 或 JSON 字段。

## 6. 编辑复杂属性

1. 在树或网格中选中某个复杂属性（如字体、数据库连接、视频片段等）；
2. 点击“编辑复杂属性”按钮，或双击该项；
3. 工具会在中间下方区域加载对应的复杂属性编辑器 Frame，例如：
   - 字体编辑器：选择字体家族、大小、样式和颜色；
   - 数据库编辑器：输入主机、端口、数据库名、用户名等；
   - 背景布局编辑器：配置背景色、图层、文本和图片叠加；
   - 视频片段编辑器：选择文件并设置起止时间、音量等。
4. 编辑完成后，点击“确定”或“应用”按钮保存修改；
5. 工具会将结果写回对应 JSON 子对象，并刷新视图。

## 7. 验证配置

1. 在工具栏或右侧区域点击“验证配置”按钮；
2. 工具会调用内部验证模块，对当前 INI + JSON 进行完整检查：
   - 必填项是否填写；
   - 数值是否在合法范围内；
   - 路径指向的文件/目录是否存在（如需）；
   - `_ref` 引用是否指向有效对象；
3. 验证结果会在底部日志区域和右侧“验证摘要”中显示：
   - 信息级（Info）：说明成功或统计信息；
   - 警告（Warning）：可自动修复或不影响核心流程的问题；
   - 错误（Error）：必须修复后才能安全使用的严重问题。
4. 用户可以根据提示定位具体项并进行修改。

## 8. 保存与备份

1. 在菜单或工具栏中点击“保存”按钮；
2. 工具会：
   - 按策略创建备份文件（例如在 `backup/` 目录生成时间戳 `.bak`）；
   - 先写 JSON，再写 INI（或按实现定义的顺序）；
   - 使用日志记录保存结果（成功或失败原因）。
3. 保存失败时，界面会提示错误原因，并建议查看日志。

## 9. 文件浏览 Tab：批量管理 INI / JSON 文件

1. 切换到主界面的“文件浏览”Tab；
2. 在顶部选择驱动器与路径：
   - 驱动器组合框用于选择磁盘；
   - 路径输入框显示当前目录，可配合“上一级”“刷新”按钮导航；
3. 左侧面板分为上下两部分：
   - 上半部（目录树）：展示目录结构，可展开/折叠导航；
   - 下半部（文件列表）：展示当前目录下的文件，通过复选框 `INI` / `JSON` / `所有` 过滤文件类型；
4. 右侧为"可编辑预览"区域：
   - 显示当前选中文件的内容；
   - 可直接在文本框中编辑，通过"保存"按钮写回磁盘。
4. 常用操作：
   - “作为工程打开”：将选中的 INI（及对应 JSON）作为工程加载，自动切回“配置编辑”Tab；
   - “新建对”：在当前目录新建一对空白 `*.ini + *.json` 文件；
   - “删除”：删除选中文件（需要确认）；
   - “在外部打开”：交给操作系统关联程序打开该文件。
5. 常用操作工具栏位于顶部或左侧文件列表上方，包括上述按钮。

## 10. 常见问题

- **Q: 打开 INI 后提示找不到 JSON 文件？**
  - 检查 INI 中 `[json_file]` 的 `file_path` 是否正确；
  - 确认路径是否为正确的相对/绝对路径；
  - 如确需新建 JSON，可在工具中使用“创建空 JSON”功能（若提供）。

- **Q: 修改配置后业务程序没有生效？**
  - 确认已点击“保存”，并检查目标 INI / JSON 是否被业务程序正确加载；
  - 某些业务程序可能需要重启或显式“重新加载配置”。

- **Q: 验证通过但仍有业务错误？**
  - ConfigBuild 只检查通用结构与基础规则；
  - 业务特定逻辑错误仍需业务程序自身的日志和校验来发现。
