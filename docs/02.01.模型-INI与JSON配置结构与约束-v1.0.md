# INI 与 JSON 配置结构与约束

> 本文描述 ConfigBuild 所编辑的“INI + JSON 双文件”配置模型，
> 作为业务程序与配置编辑器之间的契约说明。

## 1. 模型概述

- 每个“配置单元”由一对文件组成：`*.ini` + `*.json`；
- INI 负责管理**简单键值**（基本类型、文件路径、少量开关）；
- JSON 负责管理**复杂对象与数组**（背景布局、视频片段、AI 模型配置等）；
- ConfigBuild 负责在 UI 中统一展示和编辑这两个文件，并保持它们的一致性。

## 2. INI 文件约定

### 2.1 文件与编码

- 文件后缀：`.ini`；
- 编码：UTF-8（带 BOM）；
- 命名示例：`app_config.ini`。

### 2.2 关键节与键

- 约定存在一个指向 JSON 文件的节，例如：`[json_file]`：
  - `file_path`：JSON 文件相对或绝对路径；
  - 其它补充信息（如版本号）可按需扩展。

- 其它节用于简单配置分组，例如：
  - `[app_settings]`：应用级开关、基本数值；
  - `[ui_settings]`：UI 相关的颜色、尺寸、字体名等。

### 2.3 键与值

- 建议键名使用小写或小驼峰，避免空格和特殊字符；
- 值尽量为简单文本：
  - 整数 / 浮点 / 布尔：由编辑器负责转换和校验；
  - 路径 / 文件名：支持相对路径（相对于 INI 所在目录）与绝对路径。

## 3. JSON 文件约定

### 3.1 文件与编码

- 文件后缀：`.json`；
- 编码：UTF-8（无 BOM 或带 BOM，推荐统一）；
- 命名示例：`app_config.json`；
- 通常与 INI 放在同一目录，或由 INI 中的 `file_path` 指定。

### 3.2 对象结构

- 顶层为一个或多个对象，按业务需要组织：
  - 常见节点：`fonts`, `database`, `ui`, `workflows`, `videos` 等；
- 每个对象应使用：
  - `_type`：标明对象类型（对应 `TConfigType` / 复杂属性类型枚举）；
  - `_id`：对象唯一标识（在整个配置中唯一）。

### 3.3 数组与引用

- 对于列表型配置（例如视频片段列表、图层列表），使用 JSON 数组：

```json
{
  "videos": [
    { "_id": "clip1", "_type": "VideoClip", ... },
    { "_id": "clip2", "_type": "VideoClip", ... }
  ]
}
```

- 对于引用关系（例如“某一 UI 元素引用一个字体配置”）：
  - 使用 `_ref` 字段指向另一个对象的 `_id`；
  - 验证规则需要检查 `_ref` 的有效性和类型匹配。

## 4. INI 与 JSON 一致性约束

1. INI 中指向 JSON 的 `file_path` 必须能解析到存在的 JSON 文件；
2. INI 中的开关与 JSON 中的结构应保持语义一致，例如：
   - INI 中的 `enable_video=true`，应与 JSON 中是否存在 `videos` 或某字段启用状态对应；
3. JSON 中的 `_type` 必须与 `UtilsTypes.pas` 中定义的类型之一匹配；
4. JSON 中所有 `_ref` 必须能在同一配置单元内找到对应 `_id` 对象；
5. 当用户在 UI 中添加/删除某个复杂对象时：
   - 必须同时更新 JSON 对象集合；
   - 如有必要，INi 中的计数或索引也应更新（推荐尽量避免此类隐式依赖）。

## 5. 命名与路径规则

- 文件路径：
  - 默认相对于 INI 文件所在目录；
  - 如写绝对路径，则必须是有效的系统路径；
  - ConfigBuild 在保存时可提供“规范化路径”的选项（统一为相对路径）。

- 对象 `_id`：
  - 推荐使用简短的英文字符串（`font_main`, `db_default` 等）；
  - 应在整个 JSON 内保持唯一，方便引用和诊断。

## 6. 与代码实现的关系

- `ConfigManager.pas`：
  - 实现本文件中描述的 INI+JSON 加载/保存和一致性维护；
- `ConfigValidator.pas`：
  - 根据本文件中的约束规则实现自动验证；
- `UtilsTypes.pas`：
  - 定义 `_type` 字段与内部枚举之间的映射；
- `02.02.模型-复杂属性类型与编辑器映射-v1.0.md`：
  - 详细说明各复杂属性在 JSON 中的字段结构与对应编辑器。
