# 程序配置管理原则

## 1. 配置文件类型选择原则

### 1.1 INI文件使用场景
- 单层简单配置项（键值对）
- 不需要复杂数据类型或嵌套结构
- 程序启动基础配置
- 用户可能需要手动编辑的配置
- 性能敏感的频繁读取场景
- 配置项数量较少（通常少于50项）

### 1.2 JSON文件使用场景
- 多层嵌套结构的复杂配置
- 包含数组或对象集合的配置
- 需要序列化/反序列化对象的场景
- 与其他系统交互的配置
- 配置项数量多或结构复杂
- 动态生成或程序内部管理的配置

## 2. 配置文件路径规范

### 2.1 启动配置文件 (start.ini)
程序启动时，按以下顺序查找start.ini文件：

1. 可执行文件同级目录：`{ExePath}\start.ini`
2. 可执行文件同级的config子目录：`{ExePath}\config\start.ini`
3. 开发项目目录的config子目录：`{ProjectPath}\config\start.ini`

### 2.2 其他配置文件路径规则
一旦start.ini加载成功，其他配置文件的路径应遵循以下规则：

- INI配置文件存放于：`{ConfigRoot}\`目录
- JSON配置文件存放于：`{ConfigRoot}\`目录
其中`{ConfigRoot}`是以下之一：
1. start.ini中指定的配置根目录
2. 如未指定，则为start.ini所在目录

### 2.3 配置文件命名规范
- 使用小写字母和下划线
- INI文件命名：`模块名_功能.ini`（例如：`db_connection.ini`）
- JSON文件命名：`模块名_功能.json`（例如：`ui_layout.json`）

## 3. 配置加载流程

### 3.1 启动配置加载流程
1. 程序启动时，按2.1节定义的顺序查找start.ini
2. 如所有路径均未找到start.ini，创建默认配置于`{ExePath}\config\start.ini`
3. 从start.ini中加载基础配置信息：
   - 配置根目录路径
   - 日志级别和路径
   - 调试模式标志
   - 其他INI和JSON配置文件的相对路径

### 3.2 模块配置加载流程
1. 程序初始化时预加载常用配置，包括：
   - 数据库连接参数
   - 系统基本参数
   - 用户界面基本设置
2. 其他模块配置按需加载，避免一次性加载所有配置
3. 对于频繁访问的配置，实现内存缓存机制
4. 实现配置变更监听，支持热重载

## 4. 配置文件内容规范

### 4.1 INI文件内容规范
- 必须包含有意义的节(Section)分组
- 注释使用分号(;)开头
- 敏感信息（如密码）应加密存储
- 示例：
```ini
[DatabaseConnection]
Server=localhost
Port=3306
Username=appuser
; 密码使用AES加密存储
Password=EncryptedPassword123
```

### 4.2 JSON文件内容规范
- 使用4空格缩进
- 日期时间使用ISO 8601格式
- 大型JSON文件应提供注释或结构说明文档
- 敏感信息同样需要加密
- 示例：
```json
{
    "uiLayout": {
        "mainWindow": {
            "position": {
                "x": 100,
                "y": 100
            },
            "size": {
                "width": 800,
                "height": 600
            },
            "isMaximized": false
        },
        "controls": [
            {
                "id": "btnSave",
                "visible": true,
                "position": {
                    "x": 10,
                    "y": 10
                }
            }
        ]
    }
}
```

## 5. 配置管理类设计原则

### 5.1 基础接口定义
- 实现`IConfigProvider`接口作为所有配置管理类的基础
- 提供统一的读写方法和事件通知机制

### 5.2 INI配置管理器
- 实现轻量级读写器，支持标准INI操作
- 提供类型转换功能（字符串到整数、布尔值等）
- 实现配置项变更监听

### 5.3 JSON配置管理器
- 支持对象序列化和反序列化
- 实现JSON路径查询
- 提供结构验证功能

### 5.4 配置更新和持久化策略
- 延迟写入：批量更改后一次性保存
- 定时保存：对于频繁变化的配置项
- 退出保存：程序关闭时保存更改
- 实现配置版本管理和备份机制
